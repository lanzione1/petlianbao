## 二、测试环境准备

### 1. 启动服务

```bash
# 终端1：启动后端
cd apps/backend && pnpm dev

# 终端2：启动H5
cd apps/customer-h5 && pnpm dev

# 终端3：启动管理后台（可选）
cd apps/admin-web && pnpm dev
```

### 2. 服务地址

| 服务 | 地址 |
|------|------|
| 后端API | http://localhost:3000 |
| 客户H5 | http://localhost:5175 |
| 管理后台 | http://localhost:5173 |
| 商家小程序 | 微信开发者工具打开 `apps/merchant-miniapp` |

### 3. 初始化测试数据

**方式一：小程序端创建**
1. 打开微信开发者工具
2. 登录商家小程序
3. 进入"我的" → "开发者工具" → "创建测试数据"

**方式二：API创建**
```bash
# 需要先获取Token
curl -X POST http://localhost:3000/api/v1/dev/seed \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 四、详细测试步骤

### 步骤1：获取商家Token（小程序端）

1. 打开微信开发者工具
2. 导入项目 `apps/merchant-miniapp`
3. 点击"编译"按钮
4. 打开控制台（Console）
5. 输入以下命令获取Token：
   ```javascript
   wx.getStorageSync('token')
   ```
6. 复制Token值（以 `eyJ` 开头的长字符串）

**截图位置**：控制台 → Storage → token

## 五、测试场景矩阵

### 正向流程测试

| 场景 | 操作 | 预期结果 |
|------|------|----------|
| 正常预约 | H5提交 → 小程序确认 → 服务完成 | 状态流转：pending → confirmed → completed |
| 时间修改 | 小程序修改时间 | H5端显示新时间 |
| 客户取消 | H5取消预约 | 小程序收到取消通知，状态变为 cancelled_by_customer |
| 商家取消 | 小程序取消预约 | H5端状态变为 cancelled_by_merchant |

### 异常流程测试

| 场景 | 操作 | 预期结果 |
|------|------|----------|
| 重复预约 | 同一时间多次预约 | 提示时间冲突 |
| 无效Token | 使用过期Token | 提示登录失效，跳转登录 |
| 网络异常 | 断网后操作 | 显示错误提示，支持重试 |
| 参数校验 | 手机号格式错误 | 前端校验提示 |

## 八、常见问题

### Q1: Token无效或过期
**解决**：重新在小程序登录，获取新Token

### Q2: H5无法连接后端
**解决**：
1. 检查后端是否启动（http://localhost:3000/api/v1/health）
2. 检查Vite代理配置

### Q3: 小程序登录失败
**解决**：
1. 检查小程序AppID配置
2. 使用开发模式登录（自动跳过微信验证）

### Q4: 预约列表为空
**解决**：
1. 确认已创建测试数据
2. 调用 `POST /api/v1/dev/seed` 接口
