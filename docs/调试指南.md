# 微信开发者工具调试指南

## 一、导入项目

### 1.1 打开微信开发者工具

启动微信开发者工具，选择 **小程序**

### 1.2 导入项目

点击 **+** 或 **导入项目**：

| 配置项 | 值 |
|--------|------|
| 项目目录 | `E:\Projects\CWB\apps\merchant-miniapp` |
| AppID | 填写你的AppID，或选择"测试号" |
| 项目名称 | 宠联宝 |

### 1.3 使用测试号

如果没有 AppID，点击 **使用测试号**，系统会自动分配一个测试 AppID。

## 二、配置开发设置

### 2.1 右上角点击"详情"

在 **本地设置** 中勾选：

```
☑ 不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书
☑ 不校验安全域名、TLS版本以及HTTPS证书
☑ 启用调试
```

### 2.2 修改 API 地址（开发环境）

编辑 `apps/merchant-miniapp/utils/api.js`：

```javascript
const app = getApp()

const request = (options) => {
  return new Promise((resolve, reject) => {
    wx.request({
      // 开发环境使用本地地址
      url: 'http://localhost:3000/api/v1' + options.url,
      // ... 其他配置
    })
  })
}
```

或者修改 `app.js` 中的 `apiBase`：

```javascript
// apps/merchant-miniapp/app.js

App({
  globalData: {
    // 开发环境
    apiBase: 'http://localhost:3000/api/v1',
    // 生产环境
    // apiBase: 'https://api.petlianbao.com/api/v1'
  }
})
```

## 三、启动后端服务

### 3.1 启动数据库

```bash
cd E:\Projects\CWB
docker-compose up -d
```

### 3.2 启动后端

```bash
pnpm dev:backend
```

看到以下输出表示启动成功：

```
应用运行在: http://localhost:3000
API 文档: http://localhost:3000/api/v1
```

### 3.3 测试后端是否正常

浏览器访问：`http://localhost:3000/health`

应返回：

```json
{
  "status": "ok",
  "timestamp": "2025-02-20T10:00:00.000Z"
}
```

## 五、常见调试场景

### 5.1 调试登录流程

1. 打开小程序，进入首页
2. 查看 Console，应该看到登录请求
3. 如果失败，检查后端是否启动

### 5.2 调试 API 请求

在 `utils/api.js` 中添加日志：

```javascript
const request = (options) => {
  console.log('请求:', options.url, options.data)
  
  return new Promise((resolve, reject) => {
    wx.request({
      // ...
      success: (res) => {
        console.log('响应:', res.data)
        resolve(res.data)
      },
      fail: (err) => {
        console.error('请求失败:', err)
        reject(err)
      }
    })
  })
}
```

### 5.3 调试页面数据

在页面 `.js` 文件中添加：

```javascript
onLoad() {
  console.log('页面加载')
  this.loadAppointments()
}

async loadAppointments() {
  const data = await api.getTodayAppointments()
  console.log('预约数据:', data)
  this.setData({ appointments: data })
}
```

## 六、真机调试

### 6.1 预览

1. 点击工具栏 **预览** 按钮
2. 用手机微信扫描二维码
3. 在真机上查看效果

### 6.2 真机调试

1. 点击工具栏 **真机调试** 按钮
2. 扫码后可以在开发者工具中看到真机日志
3. 支持 vConsole 调试

### 6.3 注意事项

真机调试时，手机和电脑需要在同一网络，且需要：

1. 后端服务监听 `0.0.0.0` 而非 `localhost`
2. 或者使用内网穿透工具（如 ngrok）

## 七、常见问题

### Q1: "不在以下 request 合法域名列表中"

**解决方案**：
- 开发阶段：勾选"不校验合法域名"
- 生产环境：在微信公众平台配置服务器域名

### Q2: "网络请求失败"

**检查清单**：
```bash
# 1. 后端是否启动
curl http://localhost:3000/health

# 2. 数据库是否启动
docker ps

# 3. 检查 .env 配置
cat apps/backend/.env
```

### Q3: 登录失败

**检查**：
1. `project.config.json` 中的 `appid` 是否正确
2. `.env` 中的 `WECHAT_APP_ID` 和 `WECHAT_APP_SECRET` 是否配置
3. 后端日志是否有错误

### Q4: 页面空白

**检查**：
1. Console 是否有报错
2. `app.json` 中页面路径是否正确
3. 页面 `.js` 文件是否有语法错误

## 八、开发工作流

### 推荐流程

```
1. 修改代码 (VS Code)
      ↓
2. 保存文件
      ↓
3. 微信开发者工具自动刷新
      ↓
4. 查看效果 / Console 日志
      ↓
5. 如需调试后端，查看后端终端日志
```

### 同时打开两个窗口

```
窗口1: VS Code - 编写代码
窗口2: 微信开发者工具 - 查看效果
窗口3: 终端 - 运行后端、查看日志
```
